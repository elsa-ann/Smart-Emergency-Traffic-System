# Contributing

Thank you for contributing to SECS. Please follow these guidelines.

## Workflow
- Fork the repository and create a feature branch: `feature/your-feature`
- Write tests for new functionality
- Keep PRs small and focused

## Commit messages
- Use imperative style: `Add`, `Fix`, `Update`
- Reference issues: `Fixes #12`

## Code style
- Use `black` for formatting
- Run linting: `flake8 src/`

## Reporting bugs
- Create an issue with steps to reproduce, environment, and logs

## Ethics & Data
- Do not add sensitive personal data to the repo
- When adding new datasets, include source, consent, and license

## Review
- PRs should be reviewed by at least one maintainer
- Major changes to decision logic must include an ethical review note
