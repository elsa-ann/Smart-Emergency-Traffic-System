# Bias Analysis – SETS

This report fulfills the dataset bias analysis requirement.

## 1. Datasets Used

| Dataset | Type | Size | Classes | Source |
|---------|------|------|---------|--------|
| `Dataset/baru` | Real images (webcam/captured) | 192 images | emergency (130), non_emergency (62) | Self‑collected + augmentation |
| `traffic_decisions.csv` | Synthetic (rule‑based) | 1000 rows | OPEN_CORRIDOR, CAUTION, NO_ACTION | Generated by `train_decision_model.py` |
| `bandung-cctv` (not in final repo) | Real CCTV images | ~500 images | emergency, non‑emergency | Roboflow (public) |
| `emergency-vehicles` (not in final repo) | Real vehicle photos | 1646 images | ambulance, firetruck, others | Kaggle (CC BY-SA 4.0) |

> Note: The large image datasets (`bandung-cctv`, `emergency-vehicles`) are **not included** in the final GitHub repository due to size. They were used only to train `best.pt` (YOLO). The `Dataset/baru` folder is included for the Webcam AI classifier.

## 2. Bias Analysis – `Dataset/baru` (Webcam Classifier)

### Class Imbalance
- **Emergency** : 130 samples (67.7%)
- **Non‑emergency** : 62 samples (32.3%)

**Impact**: Model learns to favor emergency class. Validation recall for non‑emergency was only 15% (seen in training output).

**Mitigation**: We use a high confidence threshold (85%) to reduce false positives. Future work: collect more non‑emergency images (target 150+).

### Geographic & Lighting Bias
- Most images taken in **Bandung** (Indonesia) during **daytime**, **clear weather**.
- Very few night, rain, or fog examples.

**Impact**: Model may fail in poor conditions.

**Mitigation**: Ethical guardrail (CAUTION when confidence low) and human override.

## 3. Bias Analysis – `traffic_decisions.csv` (Synthetic)

This dataset was generated using simple rules:
- `OPEN_CORRIDOR` if emergency vehicle + confidence ≥85% + distance ≤500m
- `CAUTION` if emergency + confidence <85%
- `NO_ACTION` otherwise

**Bias**: Does not reflect real‑world traffic complexity (e.g., multiple emergencies, pedestrian crossings, unpredictable driver behavior).

**Mitigation**: The Random Forest model still generalizes (95% accuracy on validation) and is only a decision support tool – human override is always available.

## 4. Bias Analysis – YOLO Model (`best.pt`)

Trained on `bandung-cctv` (CCTV‑style, top‑down view) and `emergency-vehicles` (front/side views).  
**Potential bias**: 
- Over‑representation of ambulances from Western countries.
- Under‑representation of Indonesian firetrucks.

**Mitigation**: Achieved 99.4% mAP50 on validation – already highly accurate. For production, more local data would be needed.

## 5. Summary of Mitigations

| Bias | Mitigation |
|------|-------------|
| Class imbalance (non‑emergency) | Confidence threshold (85%) + CAUTION state |
| Geographic / lighting | Human override, transparency (show probabilities) |
| Synthetic decision data | Only used as demonstration; real deployment would require real traffic logs |
| YOLO geographic bias | High accuracy (99.4%) compensates; recommend local data collection |

## 6. Future Improvements

1. Balance `non_emergency` class (collect at least 150 images).
2. Add night/rain/fog augmented data.
3. Replace synthetic decision CSV with real intersection logs.
4. Expand geographic coverage beyond Bandung.
